Total Acres vs Tillable Acres: Logic and UI Explanation
Our LandIQ platform uses a sophisticated two-tier acreage system designed for practical agricultural land valuation. Here's how it works:

The Logic Behind Total vs Tillable Acres
Total Acreage (acreage field):

Represents the entire property size including all land types
Used for overall property boundaries and legal descriptions
Includes tillable cropland, non-tillable areas, farmsteads, roads, waterways, etc.
Tillable Acres (tillableAcres field):

Represents only the land suitable for crop production
Critical for CSR2 calculations since CSR2 only applies to cropland
Affects the precision of soil-based valuations
Typically 85-95% of total acreage on most Iowa farms
How the UI Works
Property Form Interface:

Side-by-side input fields in a responsive grid layout
Total Acreage field (left) - labeled "Total Acreage" with decimal input
Tillable Acres field (right) - labeled "Tillable Acres" with placeholder "e.g. 152"
Smart validation - tillable acres cannot exceed total acres
Mobile-responsive - stacks vertically on smaller screens
User Experience Flow:

Farmers input total property size first (familiar concept)
Then specify tillable portion (what they actually farm)
Clear visual separation with distinct labels
Intuitive number inputs with decimal support
Valuation Logic Implementation
CSR2 Calculations:

const totalCroplandValue = csr2Value * tillableAcres;
